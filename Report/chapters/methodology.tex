\chapter{Methodology}

The purpose of this chapter is to outline the approach used for designing and implementing the QRAG framework. The research method is based on the combination of classical RAG pipelines with QNLP approaches to address the problem of syntactic ambiguity in user queries. The framework is designed as a hybrid system that uses quantum approaches only when necessary to process complex queries.

\section{System Architecture Overview}

QRAG works with the help of a modular and decoupled system architecture consisting of different layers such as query processing, ambiguity detection, hybrid execution, retrieval, and response generation. All these work independently but have a defined interface for communicating with each other.

The entire process flow starts from user interaction to structural analysis of the query and then depends on the complexity, leads the process either to traditional or quantum processing route.

\section{Data Preparation and Corpus Construction}

The proposed system makes use of a diverse database containing roughly 130,000 entries stored in MongoDB. The preprocessing of this database takes place through normalization, tokenization, and embedding creation for vector-based searches.

\section{Ambiguity Detection Strategy}

A light weight POS tagger is used to find syntactic ambiguity. The algorithm recognizes the pattern such as attachment of preposition, reduced relative clauses, and coordination ambiguity. Queries with patterns matching the above are passed to quantum processing.

\section{Classical Processing Pipeline}

The classical pipeline handles low-complexity queries using dependency parsing and vector-based retrieval. It ensures low latency and efficient processing for real-time applications.

\section{Quantum Processing Pipeline}

Query encoding in the quantum pipeline uses rotation gates and dependency encoding uses entanglement realized using the CZ gate. The circuit is then run on a quantum backend to interpret which way to go.

\section{Variational Quantum Circuit Design}

The Variational Quantum Circuit (VQC) consists of alternating rotation and entanglement layers, optimized using classical optimizers such as COBYLA. This enables the system to learn complex linguistic structures.

\section{Contextual Realignment}

The system refines query interpretation before retrieval, ensuring alignment between query intent and retrieved context. This significantly improves semantic accuracy compared to traditional RAG systems.

\section{Retrieval and Generation}

The disambiguated query is used to retrieve relevant documents from MongoDB Atlas. The retrieved context is passed to an LLM for generating the final response.

\section{Evaluation Methodology}

The system is evaluated using an ambiguity-focused dataset. The evaluation metrics include:
\begin{itemize}
    \item Context Relevance (CR)
    \item Answer Faithfulness (AF)
    \item Answer Relevance (AR)
\end{itemize}

\section{Implementation Strategy}

The system is implemented using FastAPI (backend), React (frontend), MongoDB Atlas (vector storage), Neo4j (graph representation), and Qiskit (quantum execution), following a scalable cloud-native architecture.

\bigskip\noindent
The methodology combines classical efficiency with selective quantum processing to improve ambiguity resolution while maintaining performance and scalability.
